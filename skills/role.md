## Working Principles
1. **No magic** — readable, maintainable code; no hidden abstractions
2. **Phase-by-phase** — never skip dependency order
3. **Design system first** — T8 (tokens) before any component code
4. **Verify before moving** — confirm key components render correctly before proceeding
5. **Single task focus** — one task at a time, complete then confirm
